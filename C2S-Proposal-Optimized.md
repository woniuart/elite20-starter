# 赵洁_C2S_方案草案（优化版）

## 1. 我打算做什么？解决什么具体问题？

### 问题背景
- **群体痛点**：大学公共艺术选修课（每年选课学生2000+）
  - 教师：不熟悉AI工具，难以设计AI+艺术课
  - 学生：滥用AI生成作业，学术诚信风险
  - 教学：缺乏AI时代艺术教育规范

### 解决方案定位
基于Elite20三阶段体系（L1基础→L2进阶→L3创新），构建艺术学科AI教学助手矩阵：

| 助手类型 | 功能 | 对应Elite20阶段 |
|----------|------|-----------------|
| 美术鉴赏知识库 | 艺术家生平、作品分析、艺术史脉络 | L1 |
| 课前预习助手 | 知识点推送、思考题、背景资料 | L1-L2 |
| 作业批改助手 | AI检测、反馈建议、改进指导 | L2 |
| 课后复习助手 | 知识巩固、创作灵感激发 | L3 |

### 本学期核心产出
**《蒙娜丽莎》课前预习AI助手** - 完整可用的教学智能体，下学期直接投入课堂

## 2. 我凭什么觉得这个方向有效？

### 理论支撑（引用Elite20框架）
- **AI+X三维目标**：
  - L1：学生能使用AI工具理解艺术概念 ✓
  - L2：学生能借助AI完成艺术创作分析 ✓
  - L3：学生能批判性评估AI生成内容 ✓
- **双回路教学法**：
  - 外环：AI助手提供知识输入
  - 内环：学生反思+创作输出

### 实践基础
- 10年公共艺术教学经验
- 熟悉全校艺术选修课教学大纲
- 了解学生选课动机和痛点

### 实证参考
- 西亚斯学院Elite20艺术试点：学生参与度↑40%
- 同行期刊论文：AI辅助艺术教育有效性验证

## 3. 我拿到的Token/算力打算怎么花？

### 预算分配（≤30元）
| 用途 | 预算 | 占比 |
|------|------|------|
| 提示词工程迭代 | 15元 | 50% |
| API调用（知识库检索） | 10元 | 33% |
| 备用金（调试/紧急） | 5元 | 17% |

### 实验设计（3轮）

**实验1：提示词效果测试**
- 目标：验证"美术风格描述"提示词能否生成专业分析
- 指标：内容准确性评分（1-5分）
- 样本：《蒙娜丽莎》5个细节特征

**实验2：内容准确性验证**  
- 目标：AI生成内容 vs 教材/知网文献
- 指标：事实错误率 < 5%
- 对比：3个关键知识点

**实验3：学生易用性测试**
- 目标：3名学生的试用反馈
- 指标：任务完成率 > 80%，满意度 > 4分
- 方式：模拟下学期预习场景

## 4. 失败了怎么办？

### 风险清单与应对

| 风险场景 | 概率 | 应对方案 | 转化价值 |
|----------|------|----------|----------|
| AI生成内容不符合教学要求 | 中 | 人工修正+提示词迭代，保留完整迭代记录 | 提示词优化经验 |
| 工具调用失败 | 低 | 改用Claude桌面端手动完成，记录问题 | 工具选型文档 |
| 学生参与度不足 | 中 | 增加互动环节+分数激励 | 教学设计改进 |
| Token超支 | 低 | 立即停用，转入手动模式 | 成本控制案例 |

### 最低可接受成果（保底Level1）
- [x] 完成智能体环境搭建
- [x] GitHub 3次提交
- [x] 1次完整K-STAR闭环
- [ ] 4类AI助手原型（期望）/ 1个基础助手（保底）

### 迭代升级路径
```
Level1 (保底) → Level2 (核心) → Level3 (拓展)
     │              │              │
   环境搭建      《蒙娜丽莎》      4类助手矩阵
   3次提交        预习助手        课堂实战
   1次闭环        完整可用        下学期适用
```
