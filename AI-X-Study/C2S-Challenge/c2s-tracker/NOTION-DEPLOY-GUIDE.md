# Notion 模板部署指南

> 将 C2S 产出物追踪模板导入您的 Notion 工作区

---

## 方式一：手动部署（推荐）

由于 Notion API 限制，建议手动创建数据库。以下是每个模板的字段说明：

### 模板 ①：论文理解追踪

| 字段名 | 类型 | 说明 |
|--------|------|------|
| 名称 | title | 内容名称或主题 |
| 类型 | select | 摘要 / 翻译 / 评注 / 批评 / 建议 |
| 状态 | select | 未开始 / 进行中 / 待审核 / 已完成 |
| 完成度 | number | 百分比格式 |
| 质量等级 | select | 待改进 / 合格 / 良好 / 优秀 |
| 论文页码 | text | 原文引用来源 |
| 个人解读 | text | 你的分析/批评/建议 |
| 最后修改 | date | 自动日期 |

**检查清单**：
- 是否有原文引用？（页码标注）
- 是否有个人解读（非纯复述）？
- 是否区分了'论文讲的'和'我讲的'？
- 是否有具体例子或证据支撑？
- 评注是否提出了建设性意见？

---

### 模板 ②：KSTAR 行动方案追踪

| 字段名 | 类型 | 说明 |
|--------|------|------|
| 方案名称 | title | KSTAR 方案主题 |
| K-知识 | text | 需要什么信息？ |
| S-情境 | text | 具体情况/约束是什么？ |
| T-任务 | text | 真正要完成的是什么？ |
| A-行动 | text | 做了什么（包括提示词）？ |
| R-结果 | text | 发生了什么？是否符合预期？ |
| 状态 | select | 未开始 / K阶段 / S阶段 / T阶段 / A阶段 / R阶段 / 已完成 |
| 完成度 | number | 百分比格式 |
| 可执行性 | select | 待改进 / 基本可行 / 良好 / 优秀 |
| 最后修改 | date | 自动日期 |

**检查清单**：
- K - 是否明确了需要的具体信息？
- S - 是否有具体的时间/资源约束？
- T - 任务是否具体可衡量？
- A - 是否有具体的行动步骤？
- R - 结果是否有可衡量的指标？
- 整体 - 读完后能否'下学期可以试着跑一下'？

### 模板 ③：个人联结追踪

| 字段名 | 类型 | 说明 |
|--------|------|------|
| 联结主题 | title | 联结的核心主题 |
| 联结类型 | select | 专业联结 / 生活联结 / 情感联结 / 职业联结 |
| 具体事实 | text | 具体的真实生活事实（不是泛泛而谈） |
| AI关联 | text | 如何与 AI 工具/学习产生关联 |
| 独特性 | select | 模糊 / 一般 / 清晰 / 独特 |
| 状态 | select | 未开始 / 构思中 / 初稿 / 完善中 / 已完成 |
| 完成度 | number | 百分比格式 |
| 最后修改 | date | 自动日期 |

**检查清单**：

- 是否避免了泛泛而谈（如'我很感兴趣'）？

### 模板 ④：AAR 课后复盘追踪（v0.2 新增执行轨迹）

| 字段名 | 类型 | 说明 |
|--------|------|------|
| 复盘主题 | title | 复盘的主题或事件 |
| 时间线 | text | 按时间顺序的关键事件 |
| 做得好的 | text | 成功的关键因素 |
| 需改进的 | text | 需要改进的地方 |

| **AI输出摘要** | text | AI 输出的核心结论或关键信息（1-3句） |
| **协作质量** | select | 低 / 中 / 高 / 优秀 |

| 真实性 | select | 模糊 / 一般 / 清晰 / 生动 |

**检查清单**：
- 是否有具体的时间戳事件？

- 改进计划是否具体可执行？
- 读完后是否能感受到'这 4 天真实发生了什么'？
- 是否避免了 AI 凭空生成的通用性描述？
- **协作轨迹是否记录了 AI 模型和对话轮次？**
- **关键提示词是否已记录（便于复现）？**
- **协作质量评估是否真实反映 AI 帮助程度？**

## 方式二：使用脚本自动部署（需要 Notion API）

### 前置条件

1. **安装依赖**：
```bash
pip install notion-client python-dotenv
```

2. **创建 Notion 集成**：
   - 访问 https://www.notion.so/my-integrations
   - 点击 "New integration"
   - 填写名称（如 "C2S Tracker"）
   - 选择您的工作区
   - 复制 Internal Integration Token

3. **获取父级页面 ID**：
   - 在 Notion 中创建一个新页面（用于存放数据库）
   - 复制页面 URL，格式如：`https://notion.so/页面名称-xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx`
   - 复制 `xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx` 这 32 位字符作为页面 ID

4. **配置环境变量**：
   创建 `.env` 文件：
   ```
   NOTION_TOKEN=secret_xxxxxxxxxxxxxxxxxxxxxxxxxxxx
   NOTION_PAGE_ID=xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx
   ```

5. **运行部署脚本**：
   ```bash
   python deploy_to_notion.py
   ```

## 常见问题

### Q: Notion API 创建的数据库在哪里？
A: 数据库会作为子页面出现在您指定的父级页面下。如果找不到，检查集成是否已分享给该页面。

### Q: 可以直接在现有数据库上导入模板吗？
A: 可以但不建议。推荐创建新数据库测试，确认字段正确后再迁移数据。

### Q: 如何添加检查清单？
A: 检查清单是参考性内容，可以作为数据库的 description 字段，或者创建一个单独的 checklist page。
